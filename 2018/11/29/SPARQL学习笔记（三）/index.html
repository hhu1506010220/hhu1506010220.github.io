  • solution sequence modifier（修饰符）

    1.Order，按顺序排列
    2.Projection，选择某些变量
    3.Distinct，保证solution是没有重复元素的
    4.Reduced，允许消除一些不相同的solution
    5.Offset，控制solution是从solution序列的哪个地方开始选的
    6.Limit，限制solution的数量
    修饰符的出现顺序按照上面的排列。

    Order by

    ORDER BY的作用是使solution序列有序，通常是由一个表达式或者表达式+函数构成。可以选择升序ASC()或者降序DESC()，如果不加这两个函数一般来说默认是升序的。

    <运算符可以对numerics, simple literals, xsd:strings, xsd:booleans以及xsd:dateTimes这些数据结构进行排序，IRI的排序可以看作是简单字符串的排序。
    此外，SPARQL也定义了一些RDF项之间的顺序：
    1.(Lowest) no value assigned to the variable or expression in this solution.
    2.Blank nodes
    3.IRIs
    4.RDF literals
    纯文本比具有xsd:string类型的RDF literal 优先级低。
    但是SPARQL也没有把优先级定义完全，一些情况的顺序它还是没有考虑，比如像
  • “a” and “a”@en_gb (a simple literal and a literal with a language tag)
  • “a”@en and “b”@en_gb (two literals with language tags)
  • “a” and “1”^^xsd:integer (a simple literal and a literal with a supported datatype)
  • “1”^^my:integer and “2”^^my:integer (two unsupported datatypes)
  • “1”^^xsd:integer and “2”^^my:integer (a supported datatype and an unsupported datatype)
    使用ORDER BY对CONSTRUCT和DESCRIBE没有任何影响，因为只有SELECT是返回一个序列。

    Projection

    效果其实和普通的查询一样，就是说可以允许返回涉及的变量的子集的序列，也就是只返回符合条件的solution所对应的变量。

DISTINCT

DISTINCT从查询中只返回一个将相同变量绑定到相同RDF术语的solution，并且这个消除重复项是发生在LIMIT和OFFSET之后的。

REDUCED

REDUCED和DISTINCT的作用都是为了消解重复项，但是它的力度没有DISTINCT强。它返回的结果数量是随机的，也就是说不保证返回的结果是DISTINCT的，并且这个数量是使用DISTINCT返回的数量<= x <=不使用DISTINCT返回的属性。

OFFSET

OFFSET的作用就是从第几个solution开始选，组成答案序列。比如OFFSET 4那就是意味着原本的前4个solution丢弃，剩下的solution组成序列。如果OFFSET 0则没有任何影响。

LIMIT

限制返回的solution的数量，如果LIMIT 0则返回空。

  • Query Forms（查询表单）

    SPARQL有四种query form，SELECT、CONSTRUCT、ASK和DESCRIBLE。

    SELECT

    当在SELECT子句中给出变量名列表时，结果会返回特定变量及其绑定。SELECT *返回的是作用域内的所有变量（包括FLITER、MINUS右部和子查询的变量），但是它不能在有GROUP BY的查询中使用。
    结果集可以通过本地API访问，也可以序列化为JSON、XML、CSV或TSV等数据格式。



    除了绑定已有变量之外，还可以通过SELECT (表达式1 AS ?x1) (表达式2 AS ?x2)来绑定新的变量，并且在表达式2中是可以用到之前的表达式1中绑定的变量x1的。

CONSTRUCT

CONSTRUCT查询表单返回由图形模板指定的单个RDF图。结果是一个RDF图，它是通过在solution序列中获取每个solution来替换图模板中的变量，并通过union将三元组合成一个单一的RDF图。
如果所有实例去尝试产生的三元组都包含没有绑定的变量或非法的RDF构造的话，例如主语或谓词位置中的文字，那么RDF输出图中将不包含该三元组。Graph template可以包含没有变量的三元组(称为基本三元组或显式三元组)，并且这些三元组也出现在CONSTRUCT表单返回的RDF输出图中。

模板可以创建包含空白节点的RDF图。空白节点标签的作用域是当前模版。如果同一个标签在模板中出现两次，那么将为每个query solution创建一个空白节点，但对于不同的query 生成的三元组将有不同的空白节点。

对图的访问还可以用其他信息作为条件来限制。

查询的solution序列修饰符会影响CONSTRUCT查询的结果，因为CONSTRUCT是基于solution序列构造的，而solution又受修饰符的影响。

在template模板和pattern模式相同且pattern是基本图模式的情况下，CONSTRUCT有一种简写的形式（不允许使用过滤器或者复杂图模式），那就把把CONSTRUCT{xxx}和WHERE{xxx}合并在一起写成CONSTRUCT WHERE{xxx}。

ASK

可以使用ASK表单来测试query pattern是否有solution。不返回任何solution，只返回true or false。

DESCRIBE

DESCRIBE表单返回一个包含有关资源的RDF数据的RDF结果图。
DESCRIBE子句本身可以使用IRI来标识资源。最简单的DESCRIBE查询就是直接使用一个IRI，就像这样DESCRIBE <http://example.org/>
要描述的资源也可以是结果集中的被绑定的查询变量。这使得可以描述资源，无论这些资源是由IRI标识的还是由数据集中的空白节点标识的。

  • Function Definitions（常见函数）

  1. bound
    xsd:boolean BOUND (variable var)
    如果var绑定了一个值，那么函数会返回true；否则会返回false。如果变量的值是NAN或者INF，那么也会返回true。当然，我们可以在bound()前面加上!来进行取反操作。

  2. IF
    rdfTerm IF (expression1, expression2, expression3)
    此表达式会根据expression1的对错来选择取哪个值作为返回值，类似于C++中的三目运算符。如果表达式1的值为true，那么计算表达式2的值作为返回值；如果为false，则计算表达式3的值为返回值；如果表达式1或者随之计算的那个表达式中有计算结果为error的，那么整个IF语句返回error。所以说表达式2和表达式3中最多只有一个式子被计算。

  3. COALESCE
    rdfTerm COALESCE(expression, ....)
    此表达式用来返回第一个没有error的表达式的值。因为在SPARQL中，如果在计算式中有一个没有绑定值的变量，那么就会引发error。如果COALESCE中所有的表达式的值都是error，那么整个COALESCE就会返回error。

  4. NOT EXISTS and EXISTS
    xsd:boolean NOT EXISTS { pattern }和xsd:boolean EXISTS { pattern }。
    二者在之前的笔记中已经讲过，这里不作赘述。

  5. && and ||
    这里给出一张逻辑真值表：

  6. RDFterm-equal
    xsd:boolean RDFterm term1 = RDFterm term2
    如果两个RDF项是完全相同（即）的，那么返回true，具体规则如下：

  • term1 和 term2 是 RDF URI References 中定义的等价的IRI.
  • term1 和 term2 是Literal Equality 中定义的等价的literal .
  • term1 和 term2 是 Blank Nodes 中描述的相同的空白节点 .

    还有一些特殊的例子…

  1. sameTerm
    xsd:boolean sameTerm (RDF term term1, RDF term term2)
    相同RDF项返回true，否则返回false。与RDFterm-equal不同的是，sameTerm可以用来测试不支持的数据类型的非等效类型的文字。

  2. IN and NOT IN
    boolean RDFterm IN (expression, ...)
    此表达式用来检测左边的RDF项是否在右边的那个集合里。如果在返回true；如果在 并且 集合中有error生成，那么还是返回true；如果不在 并且 集合中有error生成，那么返回error；否则返回false。
    boolean RDFterm NOT IN (expression, ...)
    如果在返回false；如果在 并且 集合中有error生成，那么还是返回false；如果不在 并且 集合中有error生成，那么返回error；否则返回true。

  3. isIRI and isURI
    xsd:boolean isIRI (RDFterm term)
    xsd:boolean isURI (RDFterm term)
    如果是IRI，返回true；否则返回false。isURI是isIRI的一个等价语句。（IRI是用<>或者“”包起来的）

  4. isBlank
    xsd:boolean isBlank (RDFterm term)
    如果term是空白结点就返回true，否则返回false。

  5. isLiteral
    xsd:boolean isLiteral (RDFterm term)
    如果term是literal返回true，否则返回false。

  6. isNumeric
    是数字返回true，否则返回false。

  7. str
    simple literal STR (literal ltrl)
    simple literal STR (IRI rsrc)
    返回literal的lexical形式，返回IRI的码点形式。

  8. lang
    simple literal LANG (literal ltrl)
    返回ltrl的语言标记，如果没有就返回空。注意，RDF数据模型不包含带有空语言标记的literal。比如 FILTER ( lang(?name) = "es" )。

  9. datatype
    iri DATATYPE (literal literal)

  • 如果literal是typed literal（比如xxx^^IRI），则返回数据类型IRI。
  • 如果literal是一个简单的literal，返回xsd：String.
  • 如果literal是带有语言标记的literal，则返回rdf:langString.
    比如像这样：FILTER ( datatype(?shoeSize) = xsd:integer )

  1. IRI
    iri IRI('simple literal')
    iri IRI(xsd:string)
    iri IRI(iri)
    iri URI('simple literal’)
    iri URI(xsd:string)
    iri URI(iri)
    URI和IRI可以看作同一个东西，如果参数是IRI则原封不动得返回，如果是简单的文字或者字符串就转换成IRI再返回，其他的任何RDF项都会error。

  2. BNODE
    blank node BNODE()
    blank node BNODE(simple literal)
    blank node BNODE(xsd:string)
    BNODE函数构造一个空白节点，它不同于正在查询的数据集中的所有空白节点，，并且与调用此构造函数为其他查询解决方案创建的所有空白节点不同。如果使用blank node BNODE()形式，则每个调用都会生成一个不同的空白节点。如果使用blank node BNODE(simple literal)形式，则每个调用都会为不同的simple literal生成不同的空白节点，而对于同一个simple literal的调用则会产生相同的空白节点。

  3. STRDT
    literal STRDT(simple literal lexicalForm, IRI datatypeIRI)
    STRDT函数用来 构造有指定的词法形式和数据类型的literal。

  4. STRLANG
    STRLANG函数用来 构造有指定的词法形式和语言标记的literal。

  5. UUID
    iri UUID()
    从UUID URN scheme中返回一个新的IRI。每次调用UUID()会返回不同的UUID。它不能是“nil”UUID(全部为零)。比如<urn:uuid:b9302fb5-642e-4d3b-af19-29a8f6d894c9>。

  6. STRUUID
    simple literal STRUUID()
    返回一个字符串，该字符串是UUID的方案特定部分。也就是说，生成一个UUID，然后转换成simple literal类型，然后去掉前缀urn:uuid:。比如73cd4307-8a99-4691-a608-b5bda64fb6c1。

  • Functions on Strings（字符串函数）

    首先，字符串的类型之间有兼容性的问题，有的类型可以一起用有的则不行，见下表。

  1. STRLEN
    xsd:integer STRLEN(string literal str)
    返回xsd:integer类型的字符串的长度。

  2. SUBSTR
    string literal SUBSTR(string literal source, xsd:integer startingLoc)
    string literal SUBSTR(string literal source, xsd:integer startingLoc, xsd:integer length)
    返回以source为原串、第startingLoc位开始、长度为length的子串。首字符的下标为1。

  3. UCASE and LCASE
    string literal UCASE|LCASE (string literal str)
    把字母全部变为大写或者小写。

  4. STRSTARTS and STRENDS and CONTAINS
    xsd:boolean STRSTARTS(string literal arg1, string literal arg2)
    xsd:boolean STRENDS(string literal arg1, string literal arg2)
    xsd:boolean CONTAINS(string literal arg1, string literal arg2)
    STRSTARTS判断字符串arg1是否以arg2开头，STRENDS判断是否以arg2结尾，CONTAINS判断arg1是否包含arg2子串。

  5. STRBEFORE and STRAFTER
    literal STRBEFORE(string literal arg1, string literal arg2)
    STRBEFORE： 如果arg2的词法部分作为arg1的词法部分的子字符串出现，则函数返回arg1中arg2第一出现的位置之前的部分，类型与arg1相同（simple literal, literal language tag, xsd:string）。如果arg2是空串，那么返回空串，类型同arg1。若arg2没有在arg1中出现，则返回无类型的空串。
    STRAFTER： 如果arg2的词法部分作为arg1的词法部分的子字符串出现，则函数返回arg1中arg2第一出现的位置之后的部分，类型与arg1相同（simple literal, literal language tag, xsd:string）。如果arg2是空串，那么返回arg1。若arg2没有在arg1中出现，则返回无类型的空串。

  6. ENCODE_FOR_URI
    simple literal ENCODE_FOR_URI(string literal ltrl)
    返回ltrl的词法（内容）部分。比如| encode_for_uri("Los Angeles"@en) 返回的是 "Los%20Angeles"。

  1. CONCAT
    string literal CONCAT(string literal ltrl1 ... string literal ltrln)
    连接多个字符串。词法部分就是连接起来，类型的话，如果所有参数字符串都是xsd:string的那么最后结果也是xsd:string；如果所有参数字符串都是具有相同语言标记的纯文本那么最后结果也是这个语言标记；其他情况类型都是simple literal。

  2. langMatches
    xsd:boolean langMatches (simple literal language-tag, simple literal language-range)
    和lang的作用差不多，看语言标记是不是和指定的一样。例如：FILTER langMatches( lang(?title), "FR" )一样的话就返回true。

  3. REGEX
    xsd:boolean REGEX (string literal text, simple literal pattern)
    xsd:boolean REGEX (string literal text, simple literal pattern, simple literal flags)
    之前讲过，不再赘述。

  4. REPLACE
    string literal REPLACE (string literal arg, simple literal pattern, simple literal replacement )
    string literal REPLACE (string literal arg, simple literal pattern, simple literal replacement, simple literal flags)
    在arg中，用replacement去替换pattern，flags可设置忽略大小写等规则。

  • Functions on Numerics（数字函数）

  1. abs
    返回绝对值，abs(-1.5)返回1.5。
  2. round
    返回与参数最接近的不带小数部分的数字。比如round(2.4999)返回2.0，round(2.5)返回3.0。
  3. ceil
    向上取整。ceil(10.5)返回11.0，ceil(-10.5)返回-10.0。
  4. floor
    向下取整。floor(10.5)返回10.0，floor(-10.5)返回-11.0。
  5. RAND
    返回一个大于等于0小于1的数，返回值为double。比如"0.31221030831984886"^^xsd:double。

  • Functions on Dates and Times（时间函数）

  1. now（返回值是xsd:dateTime类型）
  2. year（返回值xsd:integer）
  3. month（返回值xsd:integer）
  4. day（返回值xsd:integer）
  5. hours（返回值xsd:integer）
  6. minutes（返回值xsd:integer）
  7. seconds（返回值xsd:integer）
  8. timezone（返回值xsd:dayTimeDuration）
  9. tz（返回simple literal类型的timezone）

  • Hash Functions（哈希函数）

    返回UTF-8形式的的MD5/SHA校验和（16进制字符串，字母为小写）。
  1. MD5
  2. SHA1
  3. SHA256
  4. SHA384
  5. SHA512
