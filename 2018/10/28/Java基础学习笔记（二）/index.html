异常处理

  • 将可能抛出异常的代码放在try{}中，处理语句{catch(xxxxxxException e){}}中编写处理异常的代码。e.printStackTrace()会打印方法的调用痕迹。并且如果出现多个异常，可以接多个catch(){}代码块。finally模块不管是否出现异常都会被执行。

  • 若try中调用了一个方法f()，并且异常会在f()中，而f()并不处理该异常而是在上一层的catch中进行处理，这时候编写f()就可以用throws xxxxxException抛出异常，然后在上一层catch住。

    • 总体有三种异常：CheckedException即必须进行处理的异常，不catch就会编译报错的那种；运行时异常，也就是除0、下标越界、空指针等异常，此类异常不是必须进行try catch的异常；错误。xxxxxException都是Exception的子类，所有异常都可以用Exception e；Exception和Error都是Throwable的子类，所有异常和错误都可以用Throwable捕捉。

  • throw和throws的区别：

    • throws出现在方法声明上，而throw通常都出现在方法体内。
    • throws表示出现异常的一种可能性，并不一定会发生这些异常；throw则是抛出了异常，执行throw则一定抛出了某个异常对象。

缓存流

以介质是硬盘为例，字节流和字符流的弊端：在每一次读写的时候，都会访问硬盘。如果读写的频率比较高的时候，其性能表现不佳。为了解决以上弊端，采用缓存流。缓存流在读取的时候，会一次性读较多的数据到缓存中，以后每一次的读取，都是在缓存中访问，直到缓存中的数据读取完毕，再到硬盘中读取。缓存流在写入数据的时候，会先把数据写入到缓存区，直到缓存区达到一定的量，才把这些数据，一起写入到硬盘中去。按照这种操作模式，就不会像字节流，字符流那样每写一个字节都访问硬盘，从而减少了IO操作。

数据流

DataInputStream，数据输入流。DataOutputStream，数据输出流。要用DataInputStream 读取一个文件，这个文件必须是由DataOutputStream写出的，否则会出现EOFException，因为DataOutputStream在写出的时候会做一些特殊标记，只有DataInputStream才能成功的读取。

对象流

对象流指的是可以直接把一个对象以流的形式传输给其他的介质，比如硬盘 一个对象以流的形式进行传输，叫做序列化。 该对象所对应的类，必须是实现Serializable接口。

集合框架

ArrayList

由于声明数组只能声明固定长度，所以会造成不够用或者浪费了不用的情况。于是ArrayList就诞生了，它的”capacity”会随着对象的增加，自动增长 只需要不断往容器里增加英雄即可，不用担心会出现数组的边界问题。ArrayList实现了接口List，常见的写法会把引用声明为接口List类型。注意：是java.util.List,而不是java.awt.List。

LinkedList

与ArrayList一样，LinkedList也实现了List接口，但特殊地是它还实现了双向链表Deque，可以方便地在头尾插入删除数据。

此外LinkedList还实现了队列Queue接口，遵循先入先出FIFO的原则。

二叉排序树

按顺序插入元素，小的或者相同的放在左边，大的放在右边。

HashMap

对于HashMap而言，key是唯一的，不可以重复的。所以，以相同的key把不同的value插入到 Map中会导致旧元素被覆盖，只留下最后插入的元素。不过，同一个对象可以作为值插入到map中，只要对应的key不一样。

HashSet

就是Set，其中的元素不能重复。其中的元素是没有顺序的，也就是说set不会自动排序，在不同条件下，元素的顺序都有可能不一样。Set不提供get()方法，所以只能通过迭代器或者增强型for循环来遍历。在HashSet的源代码中，自身并没有独立实现，而是封装了一个Map，实际上HashSet是作为Map的key而存在的。

  • 其他的几种Set：LinkedHashSet按照插入顺序；TreeSet从小到大排序。

Collection

Collection是Set、List、Queue和Deque的接口。Collection和Map之间没有关系，Collection是放一个一个对象的，Map 是放键值对的；Deque 继承 Queue，间接的继承了 Collection。

Collections

Collections是一个类，容器的工具类，就如同Arrays是数组的工具类，

  • 比较器。通过编写Comparator来制定排序规则。

    或者直接在类中实现Comparable接口，Collections.sort就有足够的信息进行排序了，也无需额外提供比较器Comparator。

其他

  • ArrayList和HashSet的区别
    • ArrayList有序，HashSet无序。
    • List中的数据可以重复，Set中的数据不能够重复。重复判断标准是: 首先看hashcode是否相同，如果hashcode不同，则认为是不同数据；如果hashcode相同，再比较equals，如果equals相同，则是相同数据，否则是不同数据。

  • ArrayList和LinkedList的区别

    • ArrayList插入删除数据慢；LinkedList插入删除数据快
    • ArrayList是顺序结构，所以定位快，指哪找哪；LinkedList是链表结构，查找某个元素需要遍历之前的所有元素，所以定位慢

  • HashMap和Hashtable的区别

    • HashMap和Hashtable都实现了Map接口，都是键值对保存数据的方式，但是HashMap可以存放 null，Hashtable不能存放null。
    • HashMap不是线程安全的类，Hashtable是线程安全的类。

  • HashMap性能卓越的原因：

    • HashMap 之所以速度快，因为他使用的是散列表，根据 key 的 hashcode 值生成数组下标（通过内存地址直接查找，没有任何判断），时间复杂度完美情况下可以达到 n1（和数组相同，但是比数组用着爽多了，但是需要多出很多内存，相当于以空间换时间）。
    • HashCode作为一个算法，简单来说它就是把一串字符串变成一串数字仅此而已。比如说hashmap的put方法，通过HashCode算法得到key对应的hashcode码，把它作为下标，定位到数组相对应位置。如果该位置没有数据，则把字符串和对象组合成键值对Entry，再创建一个LinkedList，把键值对放进LinkedList中，最后把LinkedList 保存在这个位置。如果该位置有数据，一定是一个LinkedList，则把字符串和对象组合成键值对Entry，插入到LinkedList后面。
    • 实际上是一种空间换时间的方法。
