湘潭大学程序设计竞赛题解

2018-04-26

4.26湘潭大学程序设计竞赛 4/8

比赛地址

A 时间统计

直接作差换算成秒即可，注意可能会爆ll。

B.String

首先遍历一遍6行和6列，分别记录每一行每一列的得分，并赋值给该行的元素.row和.col属性。
然后在遍历一遍6×6找行和列最多得分分别是多少记为maxrow、maxcol，然后遍历整个6×6，row==maxrow&&col==maxcol的即为所求。
因为是从行数较小的开始遍历的，所以不用考虑排序的问题。

C BOOM

水题，每给出一个长方形对应区域++即可，最后找面积最大的。

D Fibonacci进制

首先递归求斐波那契前45项（因为题目x是在int范围内，所以斐波那契求到int就够了），同时累加看看当前斐波那契数全加起来能最大多少。
因为题目要求了用最小的数，所以我们采用贪心的办法，使最高位最低，这样可以使数尽可能小。
因为题目保证所有的数都可以转化为斐波那契数之和，因此从小开始累加，找到第一个累加和大于等于n的位置。开始向下检查。
由于我们的贪心是不多用任何一个高位的数，所以如果在过程中遇到一个数，减去它还大于等于当前的n，那么这个数我们是不用的。
就这样不断往低位找，每找到一个合适的数，就用n减去这个数，再不断检验。

E 吃货

很明显这题对于m次查询，不能每次都重头来一遍，所以要记忆化一下。对美食排个序，价钱少的在前；价钱相等的话美食度高的在前。
由于限制我们能不能吃到的因素只有一个就是价钱，所以我们实际上就是把价钱当作区间，把当前的最大美食度当作值，对每个区间赋值。
以价钱为端点，端点值为该价钱的最大美食度。每次来一个查询，就先找这个点在哪个区间，这里采用upper_bound大于来找到第一个不能买的东西，然后-1就是能买的最大东西的下标，即为所求。

F maze

一个典型的BFS，难点在于对于传送门咋搞。这里采用结构体内用一个txty表示传送门出口的位置，如果txty都为-1表示该位置不是传送门，否则是。并且在bfs的时候如果传送门出口是陷阱我直接break掉。传参时附带传入走到当前位置所需要的时间。并且注意我们不一定非要走传送门，如果当前点距离终点只有两秒的距离, 则通过传送门到达终点比直接走到终点要慢1秒，所以还是要把传送门当个普通节点加进bfs。

G 又见斐波那契

矩阵快速幂。重点是找递推式子。
f[i] = 1 1 1 1 1 1 f[i-1]

f[i-1] = 1 0 0 0 0 0 f[i-2]

(i+1)^3 = 0 0 1 3 3 1 i^3

(i+1)^2 = 0 0 0 1 2 1 * i^2

i + 1 = 0 0 0 0 1 1 i

1 = 0 0 0 0 0 1 1

H 统计颜色

区间修改+区间查询，但是这个比较有趣的是你添加的数有可能是重复的。对每个区间加一个vis数组看添加的数是否出现过是不现实的，因此可以采用状态压缩的思想，因为颜色一共有60种，所以我们可以把这60种颜色看作60位，并且把60种颜色对应于2^1,2^2…2^60，然后每次update就是拿原来该区间的数和最新的这个颜色代表的数或一下，区间查询就是查一下区间的这么多数一共有几位有1。

有大神想到了更巧妙的办法：对颜色定义区间,把颜色所在区间都加进去。询问的时候遍历一遍颜色集，对于每个颜色遍历一遍它的区间即可，计数即可。
