The outcome of executing a SPARQL query is defined by a series of steps, starting from the SPARQL query as a string, turning that string into an abstract syntax form, then turning the abstract syntax into a SPARQL abstract query comprising operators from the SPARQL algebra. This abstract query is then evaluated on an RDF dataset.

RDF是一个三元组 (triple) 模型,即每一份知识可以被分解为如下形式:( subject主语, predicate谓语, object宾语)。接下来我将学习用于RDF查询的SPARQL语言。

  • Definition of SPARQL

    SPARQL查询实现的流程如下：首先我们识别SPARQLquery语句当作一个字符串，然后把这个字符串转换为抽象语法形式，然后将抽象语法转换为包含SPARQL代数运算符的SPARQL抽象查询，最后在RDF数据集上计算此抽象查询。

  • Some Definitions（基础的定义）

  1. RDF Term： 从所有 “IRI、RDF Literals、blank nodes“ 合在一起构成的集合中的一项。
  2. Simple Literal： 不包含语言标记和指定类型IRI的RDF Literal。
  3. RDF Dataset： 它可以表示为：{ G, (<u1>, G1), (<u2>, G2), . . . (<un>, Gn) }，G和Gi都是图，<ui>是IRI，并且每个ui都是不同的。G叫做默认图，(<ui>, Gi)叫做命名图。
  4. Active Graph： 数据集中用于基本图模式匹配的图。
  5. RDF Dataset Merge： 就是合并两个图数据集的意思。
  6. Query Variable： 和RDF Terms不相交的变量的集合。
  7. Triple Pattern： (RDF Terms ∪ Query Variables) x (IRIs ∪ Query Variables) x (RDF Terms ∪ Query Variables)。注意，任何主语是literal的SPARQL三元组都将无法在任何RDF图上进行匹配。
  8. Basic Graph Pattern： Triple Patterns的一个集合。
  9. Property Path： 就是从一个Triple Pattern到另一个Triple Pattern的距离（不会跨越图）。称第一个Triple Pattern的主语为路径的start，最后一个Triple Pattern的宾语为路径的end。
  10. Property Path Pattern： Triple Patterns的一个变形，属性位置可以包含属性路径表达式。
  11. Solution Sequence： solution的序列，可能是无序的。
  12. Solution Sequence Modifier： 能够起到限制solution序列作用的一些限制语句词。
  13. SPARQL Query： 是一个三元组（SPARQL algebra expression，RDF Dataset，query form）。
  14. RDF Graph： 由很多三元组构成的集合。

  • 举个查询的例子

    查询由两部分组成：SELECT子句标识要在查询结果中显示的变量variables，并且WHERE子句提供了与数据图data graph匹配的基本图形模式basic graph pattern。本例中的basic graph pattern由一个带有单个变量(?title)的 triple pattern组成。可以解释为，select选择要显示的变量，where表示这个变量在哪个位置上，并且对query的变量有什么要求。运行得到的答案也就是"SPARQL Tutorial".

  • Query Result（结果集）

    查询的结果是一个solution sequence，对应于查询的图模式与数据匹配的方式。查询可能有零、一个或多个solutions。再举个例子，当数据很多时，相当于现在有a b c三个item，ab都有name和mbox属性，c只有mbox属性。然后查询语句的意思是 显示name和mbox，查询的方式是设置两个限制，一个是对于?name的，一个是对于?mbox的。也就是每一条?x ?y ?name都说明了把?name和哪个变量绑定在一起。没有?的地方就说明限定了那个变量的值必须是那个常量。而?x的作用就是代替变量，表示那个位置是一个变量，但我们没有select出来它。

  • DataType（数据类型）

    RDF有多种类型，如下：
    1.IRI
    2.literal
    3.lexical form
    4.plain literal
    5.language tag
    6.typed literal
    7.datatype IRI
    8.blank node
    比如“cat”@en表示具有词汇形式的cat，”42”^^xsd:integer表示datatype为http://www.w3.org/2001/XMLSchema#integer的integer。但是也很产生一些很相似的数据类型，如果要匹配的话要完全匹配才行。比如”cat”和”cat”@en是不同的两个类型，如果光写“cat”你是匹配不到值为”cat”@en的数据的；但是integer是比较特殊的，比如 "42"^^<http://www.w3.org/2001/XMLSchema#integer> 和 42 是相同的（可以理解为缩写）.

  • Query form（查询）

    The SELECT query form返回的是变量绑定。The CONSTRUCT query form 返回的是一个基于查询pattern生成的RDF三元组（triple pattern）。实际上相当于把WHERE中的三元组（xyz）和CONSTRUCT中的pattern（xy’z）绑定起来，也就是把WHERE中的y的值赋值给了CONSTRUCT中的y’。注意，WHERE并不是必须写的，也可以把WHERE省略只留{}也起到了一样的作用。

  • FILTER（过滤器）

    图模式匹配产生了一些 solution sequence，每个solution中都有一组变量到RDF term的绑定。SPARQL会通过FILTER表达式来过滤solution(FILTER表达式一般在where里，放的位置无所谓，只要在同一个pattern里就行)。

  1. Regex:像regex这样的FILTER，只用来匹配字符串literal，也可以通过str函数来匹配其他literal的lexical form（词法形式）。FILTER regex有三个参数位(变量名，正则规则pattern，模式位(比如i表示不区分大小写))如FILTER regex(?title, “^SPARQL”)表示对?title的限制为字符串必须是以SPARQL开头的。
  2. 数值限制: FILTER (?price < 30.5)可以直接限制算数表达式的数值。
  3. 其他:除了数字类型之外，SPARQL还支持xsd:string, xsd:boolean，xsd:dateTime等数据类型的直接限制。

  • 什么是IRI

    IRI，国际化资源标识符，是URI的generalization，并且和URI和URL完全兼容。一个前缀名映射到一个IRI，其中‘<’‘>’不构成iri的一部分。注意，带有以下字符”<”, “>”, ‘“‘ (double quote), space, “{“, “}”, “|”, “\”, “^”, “·”(英文状态下的·)的 RDF URI references 都不是IRI，因为SPARQL语言无法解析这些字符。
    BASE关键字定义了另一种解析IRI的方式，它用<>来代替BASE所指的iri，比如<a>和BASE <http://example.org/book/> <a>的含义都是http://example.org/book/book1。下面的三个写法表示的是同一个iri：

  • Prefixed name（前缀名）

    A prefixed name is a prefix label and a local part, separated by a colon “:“，并且通过把prefix指代的前缀和local part连接起来，形成一个新的iri。注意prefix label和local part可为空。local name的命名规则是，允许前导数字、允许带有非字母数字字符。

  • Syntax for Literals（词汇）

    文字的一般语法是一个字符串(括在双引号中，"..."或者单引号，'...')，使用可选的语言标记(@开头)或可选的datatype IRI或prefixed name(^^开头)。为了方便起见，整数可以直接书写(没有引号和显式数据类型iri)，并被解释为数据类型xsd：integer的类型化文字；有‘.’但没有指数的十进制数被解释为xsd:decimal；而带指数的数字被解释为xsd：double。类型值xsd：boolean也可以写成true 或 false。而对某句话的引用可以用"""xxx"""或者'''xxx'''来表示。下面我们来举些例子：

  • Blank Node（空白节点）

    查询结果可以包含空节点，形式表达为“_：”，后面为空白节点标签。比如“-:a”表示空白节点a。Blank node labels are scoped to a result set or, for the CONSTRUCT query form, the result graph。结果集中的空白标签可以相同（表示solutions中的RDF terms是相同的）或者不同（表示RDF terms不同）。并且数据集中的-:a和结果集中的-:a并没有什么联系，也就是说查询中的空白节点标签不会引用数据中的特定空白节点。空白节点在图形模式中，它充当一个新的变量，而不是特指查询数据中某个空白节点的引用，因此它的声明是不具有特定意义的。
    Blank node可以用_:xxx声明（一般形式，表示这个空白结点我们暂时给它个名字叫做xxx，但是别的地方无法引用这个xxx），也可以用[]声明（当且仅当这个blank node是查询语句的唯一所想要指代的空白节点）。一个blank node被用来组成一个三元组（triple pattern）。The same blank node label cannot be used in two different basic graph patterns in the same query.在同一个查询下面不能出现label相同的空白节点。并且 [:p "v"]这种形式的构造也可以用在triple pattern中，表示创造了一个空白节点，并且它是所有包含的predicate-object pairs（谓语-宾语对）的主语，而生成的这个[:p "v"]又可以作为主语或者宾语去构造新的三元组[:p "v"] :q "w"。

  • Triple Pattern（三元组模式）

    Triple Pattern的组成形式是( subject主语 , predicate谓语 , object宾语)。
    如果三元组有共同的主语，那么主语可以只声明一次，并且三元组之间用;隔开。

    如果三元组共同的主语和谓语，那么三元组可以合起来写，并且宾语用，隔开。

    上述两个规则合在一起用，就可以一句话声明多个三元组。

  • Graph Pattern（图模式）

    SPARQL是一种基于图模式匹配的查询语言，我们可以通过把一些简单的模式组合在一起，形成复杂的图模式。比如以下几种：
    1.Basic Graph Patterns，where a set of triple patterns must match。
    2.Group Graph Patterns，where a set of graph patterns must all match。
    3.Optional Graph patterns，where additional patterns may extend the solution。
    4.Alternative Graph Pattern，where two or more possible patterns are tried。
    5.Patterns on Named Graphs，where patterns are matched against named graphs。
    这里我们将介绍其中的两种： basic graph patterns 和 group graph pattern。
    Basic graph pattern是三元组（triple pattern）的集合，SPARQL图模式匹配就是根据基本图匹配的结果来定义的。可以理解为，三元组序列+过滤器，就组成了一个基本图模式。注意，一个空白标签只能在一个基本图模式中使用。SPARQL在进行图匹配时是基于子图匹配的思想来实现的。
    Group graph pattern组图模式用{}分割，其基本组成单元是一个个的basic graph pattern。可以理解为，在{ { b } { c } } 这样的模式中，b c都是基本图模式，而包含有{b}{c}的这个{}就是一个组图模式，它的结果其实和{b c}是一样的。而{}这样的没有任何元素的模式叫做Empty Group Pattern空组模式，它会匹配到一个没有任何信息的空图（也就是说没有绑定任何变量）。

  • Optional Values（附加值）

    Basic graph pattern在进行查询时，整个query pattern必须匹配才能找到solution。对于一个只包含group graph pattern的query，并且这些组图模式包含至少一个基本图模式，这种query所对应的solution会使得每个变量都绑定到solution中的一个RDF项。然而，我们不可能假定所有的RDF图都是正则完全的结构，于是我们提出了optional选项，它使得信息（这些信息部分匹配query pattern）可以被添加进solution但是不会绑定任何变量。也就是说，If the optional part does not match, it creates no bindings but does not eliminate the solution.使用的形式为 _pattern_ OPTIONAL { _pattern_ }。使用可选项的结果就是，要么，这个可选图模式与一个图相匹配，然后定义并且绑定到某个或很多个solution上面（附加条件成立）；或者不添加任何附加绑定、保持solution不变（附加条件不成立）。举个例子：

  • UNION（联合）

    SPARQL的UNION和Mysql差不多，就是UNION内的任何一项如果能匹配上，那么就匹配成功。如果同时有多个都可以匹配，那么会返回所有的可匹配的solution。

    不同UNION子项中绑定的变量可以相同，也可以不相同。

    UNION子项实际为一些graph pattern，每个graph pattern里可以有一个或多个三元组。
