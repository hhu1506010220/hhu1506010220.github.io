HDOJ 1285 确定比赛名次（拓扑排序）

2016-10-31

确定比赛名次

Time Limit: 2000/1000 MS (Java/Others) Memory Limit: 65536/32768 K (Java/Others)
Total Submission(s): 23293 Accepted Submission(s): 9431
Problem Description

有N个比赛队（1<=N<=500），编号依次为1，2，3，。。。。，N进行比赛，比赛结束后，裁判委员会要将所有参赛队伍从前往后依次排名，但现在裁判委员会不能直接获得每个队的比赛成绩，只知道每场比赛的结果，即P1赢P2，用P1，P2表示，排名时P1在P2之前。现在请你编程序确定排名。

Input

输入有若干组，每组中的第一行为二个数N（1<=N<=500），M；其中N表示队伍的个数，M表示接着有M行的输入数据。接下来的M行数据中，每行也有两个整数P1，P2表示即P1队赢了P2队。

Output

给出一个符合要求的排名。输出时队伍号之间有空格，最后一名后面没有空格。
其他说明：符合条件的排名可能不是唯一的，此时要求输出时编号小的队伍在前；输入数据保证是正确的，即输入数据确保一定能有一个符合要求的排名。

题意：

首先！！我严重的觉得这道题不科学！！！尼玛，根本不符合平时比赛的排名规则啊。。。

我一开始以为是按积分排序的，谁赢了谁就分多谁就在前面。。结果题目竟然说只要A赢了B那A一定在B前面。。真的是智障啊，要这么说，世界冠军输了一次他就永远再也拿不到世界冠军了？？笑话。

害得我读错题老半天。。。虽说题目说不会出现1胜2，2胜3，胜1的情况，但还是真的好坑啊啊啊啊啊….

说正事。这道题的题意就是排个序，每一场赢得人必定排在输的人前面。。。然后就用到了拓扑排序。。好像很厉害的样子。。。

先统计哪些队伍输了比赛，输了几次（用一个一维数组即可表示，数组序号表示队伍号，每输一次，数组对应的元素加一），然后查找那些没有输过的的队伍，让他们入队，再把那些输给这只队的队伍的失败次数减一，接着再查找失败次数为零的队伍，让他们入队…重复前面的步骤，直到把所有队伍都标记为已记录。
