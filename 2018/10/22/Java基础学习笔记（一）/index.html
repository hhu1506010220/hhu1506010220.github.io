环境变量配置

  • Java跨平台的原理：不同的操作系统有不同的Java虚拟机（JVM），Java程序运行在JVM中。
  • JRE：Java runtime envirornment。Java运行环境，里面包括了JVM和核心类库。
  • JDK：Java development Kit.开发工具包。
  • JVM运行的是.class字节码文件。编译过程是用javac实现的。先javac xx.java，再java xx来加载到虚拟机中运行。
  • JVM内存区划分：
    • 堆：存放对象（凡是new出来的东西都看作是对象）；栈：存储变量

变量

  • bit1一位，byte8位，short16位，char16位，int32位，long64位，float32位，double64位。byte的范围[-128,127]。1字节byte=8bit。
  • 2进制0b开头，8进制0开头，16进制0x或者0X开头。Java7开始可以在数字之间使用下划线。直接写1123代表int，longlong后面要加1123LL。直接写3.14代表double，float后面要加3.14f，并且double不能强转float。‘\u0041’表示用16进制整数表示字符。Java中不能把01赋值给boolean类型变量。
  • 对于变量的隐式转换，只要等式右边的类型的范围可以包含在等式右边的类型的范围中（低精度->高精度），就可以自动隐式转换，否则不能。
  • 如果任何运算单元的长度都不超过int,那么运算结果就按照int来计算。如byte a = 1; byte b= 2; a+b 的结果为int 类型。

运算操作符

  • Scanner类输入。

  • 长路 和 短路

    • 长路与&，不论两侧表达式的真假 都会被运算；短路与&&，若第一个表达式为false，第二个就会被短路。
    • 长路或|，不论两侧表达式的真假 都会被运算；短路或||，若第一个表达式为true，第二个就会被短路。

  • 异或^，不同为真，相同为假。

  • 通过Integer.toBinaryString()方法，可以将一个十进制整数转换为一个二进制字符串。

  • |对二进制表示的每一位进行或运算；&对二进制表示的每一位进行与运算；^对二进制表示的每一位进行异或运算；~对二进制表示的每一位取反。

  • >>带符号右移，最左边正数补0负数补1；>>>无符号右移，负数变正数。

  • “==”：两个对象地址完全一致（即两个引用指向同一个对象）时返回true，否则返回false。equals()方法，对于非String类的对象，和双等号一个效果，对String类对象可以直接利用equals()方法比较字面量是否相同（区分大小写），equalsIgnoreCase()忽略大小写，若字符串相同则返回true否则返回false。

  • switch可以使用byte,short,int,char,String(Java从1.7开始支持switch用String),enum。

数组

  • java创建数组：int[] a = new int[5];或者int[] a = new int[]{100,102,444,836,3236};或者int[] b = {100,102,444,836,3236};

  • a.length()可访问长度，数组的长度是不可变的，一旦分配好空间，是多长，就多长。

类和对象

  • java里对于对象的”XX h = new XX()”，new XX()产生的是一个对象或者叫实例，h叫做引用，多个引用可以指向同一个对象，一个引用只能指向一个对象。

  • 构造方法：public 类名(){}。一旦提供了一个有参的构造方法，又没有显式的提供一个无参的构造方法，那么默认的无参的构造方法就不存在了。this代表当前类，可以通过this访问属性，同时可用this()调用构造方法。在类的方法里，this.静态变量名 和 静态变量名 等效。

  • 继承：public class 类名 extends 父类名。当实例化一个子类时，子类和父类的构造方法都会被调用，并且是父类的构造方法先被调用，子类构造方法会默认调用父类的无参的构造方法。与this的用法相似，super代表父类。super()调用父类构造方法，super.类属性/super.类方法()。

  • private、package、protected、public的区别：属性通常使用private封装起来，属性通常使用private封装起来，会被子类继承的方法，通常使用protected，package用的不多。

  • 当一个属性/方法被static修饰的时候，就叫做类属性/方法，又叫做静态属性/方法。当一个属性/方法被声明成类属性/方法，那么所有的对象，都共享一个值，类名.类属性/方法，不需要对象的存在。对象属性/方法： 又叫实例属性/方法，非静态属性/方法。对象属性初始化有3种：1. 声明该属性的时候初始化 2. 构造方法中初始化 3. 初始化块{}。类属性初始化有2种：1. 声明该属性的时候初始化 2. 静态初始化块。

  • import java.*只会导入下面的直接的子包，而不会导入子包的子包

  • 单例模式，又叫做 Singleton模式，指的是一个类，在一个JVM里，只有一个实例存在。通过私有化其构造方法，使得外部无法通过new得到新的实例。

    • 下面的方法通过提供一个public static的getInstance方法，外部调用者只startsWith能通过该方法获取对象，而且每一次都是获取同一个对象，从而达到单例的目的。 这种单例模式又叫做饿汉式单例模式，无论如何都会创建一个实例。

    • 另一种是把new对象写到getInstance里，且只能new一次，这种模式叫懒汉式单例模式。

    • 二者的区别：

      • 饿汉式是立即加载的方式，无论是否会用到这个对象，都会加载。 如果在构造方法里写了性能消耗较大，占时较久的代码，比如建立与数据库的连接，那么就会在启动的时候感觉稍微有些卡顿。
      • 懒汉式，是延迟加载的方式，只有使用的时候才会加载。 并且有线程安全的考量。 使用懒汉式，在启动的时候，会感觉到比饿汉式略快，因为并没有做对象的实例化。 但是在第一次调用的时候，会进行实例化操作，感觉上就略慢。
      • 看业务需求，如果业务上允许有比较充分的启动和初始化时间，就使用饿汉式，否则就使用懒汉式

    • 面试常问：

      • 什么是单例模式？
      • 答案三要素：
        • 构造方法私有化
        • 实例化对象是静态属性
        • public static的 getInstance方法，返回静态实例化对象startsWith

  • 枚举类型。枚举enum是一种特殊的类(还是类)，使用枚举可以很方便的定义常量。

接口与继承

接口

public class 类名 implenments 接口名。接口内的方法一般都是一个空方法，具体实现应该在后续类中进行override重写。实现某个接口，就相当于承诺了某种约定。如果一个类实现了接口，那么该子类必须实现父接口的所有方法。但是在JDK8中添加了一个新特性，指的是接口也可以提供具体方法了，具体的就是添加了default默认方法，该方法可以有具体的实现，并且在实现该接口的类中可以不重写该方法。假设没有默认方法这种机制，那么如果要为该接口增加一个新的方法,那么所有实现了该接口的类，都需要做改动。但是引入了默认方法后，原来的类，不需要做任何改动，并且还能得到这个默认方法。

  • 引用：引用a instanceof 类名A 判断一个引用a所指向的对象是否是A类或者A的子类。
  • 重写：override （需方法名、参数完全一致）是子类覆盖父类的对象方法，重写后就不会执行父类的该方法了，只能执行当前类的该方法。
  • 多态：要实现类的多态，需要如下条件 1. 父类（接口）引用指向子类对象 2. 调用的方法有重写。
  • 隐藏：是子类隐藏父类的类方法。
    • 举例：假如父类A有一个类方法X()，子类B继承了父类并且也有一个不一样的类方法X()。
    • 若是 A a = new B(); a.X(); 则会调用A的X()。
    • 若是 B a = new B(); a.X(); 则会调用B的X()。
    • 所以对于隐藏，调用哪个类方法，与引用指向哪个对象无关，只和引用的类型有关系。

类型转换

所有子类可以转换成其父类。父类转子类要具体分析，要看两个类最后的引用是否是父子类关系，若是则可以转换。没有继承关系的两个类，不能互相转换，即使他们继承自同一个父类。比如看A是否能转换成，就看B的属性和方法是不是A中都有。若是则可以转换。转换后保持原类的属性、方法。

  • 声明一个类时，默认是继承了Object。当一个对象没有任何引用指向它时，它就满足垃圾回收的条件，当它被垃圾回收时，它的finalize()就会被调用。finalize()不是开发人员主动调用的方法，而是由虚拟机JVM调用的。

  • 关键词 final ： 当类被修饰成final的时候，表示类不能够被继承，其子类会出现编译错误。当方法修饰成final时，该方法在子类中不能被重写。final修饰基本类型变量时，表示该变量只有一次赋值机会。引用被修饰成final，表示该引用只有1次指向对象的机会。在匿名类中使用外部的局部变量，外部的局部变量必须修饰为final 。

抽象类

在类中声明一个方法，这个方法没有实现体，是一个“空”方法，这样的方法就叫抽象方法，使用修饰符“abstract”，当一个类有抽象方法的时候，该类必须被声明为抽象类。当然，抽象类不一定非得有抽象方法。一旦一个类被声明为抽象类，就不能被直接实例化。如果一个类继承了抽象类，那么该子类必须实现抽象类的所有抽象方法。

  • 抽象类和接口的区别：
    • 区别1：子类只能继承一个抽象类，不能继承多个；子类可以实现多个接口。
    • 区别2：抽象类的属性可以是 public/protected/package/private {static} {final}；但是接口中声明的属性只能是public static final的，就算没有显示的声明也默认该类型。

内部类

内部类分为四种，非静态内部类、静态内部类、匿名类、本地类

  • 非静态内部类。语法：外部类的实例.new 内部类()，可以直接访问外部类的private属性。
  • 静态内部类。语法：new 外部类.静态内部类()，不需要一个外部类的实例为基础，因此不可以访问外部类的实例属性和方法，除了可以访问外部类的私有静态成员外，静态内部类和普通类没什么大的区别。
  • 匿名类。声明一个类的同时实例化它 ，形如 A a = new A(){this.name = "111";}。
  • 本地类。本地类可以理解为有名字的匿名类。内部类与匿名类不一样的是，内部类必须声明在成员的位置，即与属性和方法平等的位置。 本地类和匿名类一样，直接声明在代码块里面，可以是主方法，for循环里等等地方。

数字与字符串

  • 所有基本类型都有对应的类类型。比如Number及其子类Byte，Short，Integer，Long，Float，Double。可以使用Interger integer = new Integer(i)将基本类型转换成封装类，也可以通过等号’=’自动转换（叫做装箱），i = integer.intValue();将封装类型转换成基本类型，也可以通过等号’=’自动转换（叫做拆箱）。Integer.MAX_VALUE 表示int的最大值。

  • String.valueof(integer)和(integer.toString())用来转换数字到字符串；Integer.parseInt(str)字符串转数字。

  • Character类的一些方法：isLetter()判断是否为字母，isDigit()判断是否为数字，isWhitespace()是否是空白，isLowerCase()是否大写，toUpperCase()转换为大写，toString()转换为字符串。

  • 在java中，字符串是一个final类，所以我们见到的字符串都是对象。常见创建字符串手段：1. 每当有一个字面值出现的时候，虚拟机就会创建一个字符串；2. 调用String的构造方法创建一个字符串对象；3. 通过+加号进行字符串拼接也会创建新的字符串对象。String对象一旦创建之后就是不可改变的，所有的改变都会生成一个新的字符串对象。

  • StringBuffer是可变长的字符串。和String内部是一个字符数组一样，StringBuffer也维护了一个字符数组。 但是这个字符数组留有冗余长度。例如new StringBuffer(“the”)，其内部的字符数组的长度是19，而不是3，这样调用插入和追加，在现成的数组的基础上就可以完成了。 如果追加的长度超过了19，就会分配一个新的数组，长度比原来多一些，把原来的数据复制到新的数组中，看上去数组长度就变长了。值得一提的是，StringBuffer的性能较之String提升很大。

    • StringBuffer的扩容，可以根据插入字符串的长度和已经存在的内容的长度计算出一个新的容量，然后根据这个容量创建一个新的数组，接着把原来的数组的内容复制到这个新数组中来，并让用于存储字符串的字符数组引用指向新的数组，从而完成扩容。

日期与时间

  • Date类，java.util.Date。所有的数据类型，无论是整数、布尔、浮点数、字符串还是日期，最后都需要以数字的形式表现出来。特殊地，0代表Java中的时间原点，其对应的日期是1970年1月1日 8点0分0秒 。

  • new Date(); 创建一个时间对象，可传一个long作为参数进行初始化，date.toString()表示当前时间，getTime() 得到表示具体0过去多少毫秒的long型的整数，与System.currentTimeMillis()结果一样。

  • 日期的格式化输出可利用SimpleDateFormat类。

  • Calendar类即日历类，常用于进行“翻日历”，比如下个月的今天是什么时候。
